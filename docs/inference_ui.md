# AtCoder-RAG 推論パイプライン・UI設計仕様書

## 1. システム概要
構築済みのChromaDB（ベクトルDB）を活用し、StreamlitのWeb UIからユーザーの曖昧な検索クエリを受け取り、関連する過去問を提示するシステム。APIリソース（特にLLM生成枠）の節約と、ノイズデータの柔軟なハンドリングを両立する設計とする。

## 2. 画面レイアウト設計 (Streamlit UI)
画面は左側の「サイドバー（検索設定）」と、右側の「メインエリア（検索実行・結果表示）」に分割する。

### 2.1 サイドバー（設定・フィルタ）
* **AIクエリ拡張トグル**
    * **UI:** チェックボックスまたはトグルスイッチ
    * **初期値:** `ON` (True)
    * **動作:** ONの場合は検索前にGemini APIを1回叩いてキーワードを拡張する。OFFの場合は入力文字列をそのままベクトル化する。
* **難易度（Difficulty）フィルタ**
    * **UI:** チェックボックス（「Difficultyで絞り込む」） ＋ 数値入力フィールド（最小値・最大値）
    * **初期値:** 絞り込み `ON`、最小値 `300`、最大値 `700`
    * **動作:** スライダーによるハードリミット（0〜4000など）は設けず、任意の数値を入力可能とする。マイナス値や外れ値にも対応可能にするため。
* **検索件数（トップK）設定**
    * **UI:** 数値入力フィールド
    * **初期値:** `5`

### 2.2 メインエリア（検索と結果）
* **検索インターフェース**
    * **UI:** テキスト入力フィールド ＋ 「検索」ボタン
* **検索結果表示（カード形式）**
    * 上位から類似度（距離）順に、以下の項目を表示する。
        1. **問題タイトル**
        2. **URL**（ハイパーリンク）
        3. **Difficulty**
        4. **抽出アルゴリズム・キーワード**（DBのメタデータから取得し、タグや箇条書きで表示）
        5. **類似度（距離スコア）**（オプション情報として小さく併記）
        6. **問題文（※TBD）**（DBに本文がない場合は、ローカルの中間JSONから読み込む処理を追加する）
    * **除外項目:** LLMによる「200字要約」はノイズ防止のためUI上には表示しない。

## 3. 処理フロー
ユーザーが「検索」ボタンを押下してからのバックエンド処理（`retriever.py`）のフローを定義する。

1. **入力の受け取り**
    * クエリ文字列、AI拡張のON/OFF、DiffフィルタのON/OFF（およびMin/Max値）、取得件数を受け取る。
2. **クエリのベクトル化準備（拡張機能）**
    * **【AI拡張がONの場合】** 入力文字列をGemini APIに送信し、「想定されるアルゴリズムとキーワード」のみを列挙した短い文字列を生成させる。（API消費：1回）
    * **【AI拡張がOFFの場合】** 入力文字列をそのまま使用する。（API消費：0回）
3. **ベクトル化 (Embedding)**
    * 決定した文字列をEmbeddingモデル（`models/embedding-001`）に渡し、ベクトルを取得する。
4. **ベクトル検索 (ChromaDB)**
    * 取得したベクトルを用いてChromaDBの `collection.query()` を実行。
    * **【DiffフィルタがONの場合】** `where={"difficulty": {"$gte": min_val, "$lte": max_val}}` を引数に付与する。
        * *仕様メモ:* ChromaDBの仕様上、この条件を指定すると `difficulty` が未設定（Null/None）のデータは自動的に除外される。
    * **【DiffフィルタがOFFの場合】** `where` 条件を指定せず、全データを対象に検索する。
5. **結果の整形と返却**
    * ChromaDBから返却された `ids`, `distances`（類似度）, `metadatas`, `documents` を紐付け、UI表示用のリスト構造に整形してStreamlitへ返す。

## 4. モジュール・関数仕様案
`src/retriever.py` に実装する主要関数のインターフェース。

| 関数名 | 役割 | 引数 | 戻り値 |
| :--- | :--- | :--- | :--- |
| `expand_query_with_llm` | AIクエリ拡張の実行 | `query: str` | `str` (拡張されたキーワード群) |
| `search_similar_problems` | ChromaDB検索の実行 | `query_vector: list[float]`, `top_k: int`, `min_diff: int \| None`, `max_diff: int \| None` | `list[dict]` (検索結果のリスト) |

*※ChromaDBのクライアント初期化コードやEmbedding関数は、DB構築時の `embedding_db.py` の資産を活用するかは実装時に決定する。*

## 5. クエリ拡張用プロンプト仕様
`expand_query_with_llm` 関数で使用するシステムプロンプトのベース案。出力を最小限に抑え、速度と精度を担保する。

> **システムプロンプト:**
> ユーザーから競技プログラミング（AtCoder）の問題を探すための曖昧な検索キーワードが与えられます。
> この検索語句から、関連性の高い「アルゴリズム名」「データ構造」「解法に関連する専門用語」を5〜10個程度予測し、スペース区切りで出力してください。
> 文章による解説や要約は一切含めず、キーワードのみを出力してください。
